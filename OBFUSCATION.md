# Инструкция по обфускации кода

## Установка инструмента обфускации

```bash
npm install -g javascript-obfuscator
```

Или локально:
```bash
npm install --save-dev javascript-obfuscator
```

## Запуск обфускации

### Полная обфускация (рекомендуется):
```bash
npm run obfuscate:full
```

### Простая обфускация:
```bash
npm run obfuscate
```

## Что делает обфускация

1. **Удаляет все console.log/warn/error** - код уже очищен от логирования
2. **Обфусцирует имена переменных** - делает код нечитаемым
3. **Добавляет защиту от отладки** - блокирует DevTools
4. **Шифрует строки** - использует base64 кодирование
5. **Добавляет мертвый код** - усложняет анализ
6. **Self-defending** - защита от деобфускации

## Использование обфусцированных файлов

После обфускации обновите `index.html`:

```html
<!-- Замените -->
<script src="/js/app.js"></script>
<script src="/js/token-loader.js"></script>

<!-- На -->
<script src="/js/app.obf.js"></script>
<script src="/js/token-loader.obf.js"></script>
```

## Важно

- Обфусцированные файлы работают медленнее (на 10-20%)
- Размер файлов увеличится в 2-3 раза
- Всегда сохраняйте оригинальные файлы для разработки
- Тестируйте обфусцированные файлы перед деплоем

## Дополнительная защита

Для максимальной защиты можно:
1. Использовать WebAssembly для критических функций
2. Использовать серверный рендеринг для чувствительных данных
3. Добавить проверки целостности кода
4. Использовать CSP (Content Security Policy)
